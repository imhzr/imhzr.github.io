部署自定义kindle推送 KindleEar 到 GAE

参考：http://kindlefere.com/post/19.html
https://github.com/cdhigh/KindleEar

自从入手kindle后，一直想好好利用。
在kindle 伴侣网站看到一篇关于部署自定义kindle推送的文章，于是决定自己折腾个，部署过程中踩的坑如下：

整个过程中最多的坑就在翻墙那，一开始参考 kindle伴侣网站上的教程一步一步部署结果，一直卡在那动初步怀疑是墙的问题，lantern 不行 换 Shadowsocks （Shadowsocks还根据网站提供的免费SS账号换了好几个服务器）结果还是报错，经过google 还是墙的问题，已经全局代理了啊，亲！！

阿西吧！接着换翻墙工具 赛风？ 不行！ VPNgate 太坑了好线路链接不上，而且经常断！！好不容易连接上一个 试了试部署还是不行！

放弃？ 嗯，放弃这种方式，采用 github 上提供的部署工具 KindleEar-Uploader 部署！结果一直要我输入邮箱和密码而且老是提示邮箱密码没错，好吧，检查下邮箱密码，麻痹没错啊，google ，继续google 最终确认是工具的问题应该是版本太旧了，听说最近GAE的SDK更新了，好吧，害我翻墙下载了老半天结果还是不行！！

继续各种折腾，还是怀疑是VPN的问题

想起以前用使用过的老牌付费VPN MXVPN

下载，注册，免费使用登录上去就有50M流量 部署下程序足够了~~

C:\Windows\System32>cd C:\Program Files (x86)\Google\google_appengine
采用最原始的部署方式部署，OK 啦 哈哈，部署过程如下！！

出错了根据提示回滚下

回滚没有报错，哈哈，继续跑~~

终于动起来了，继续。。

OK, 访问 http://hzrkindle.appspot.com/ 进行配置 (此处用我mxvpn 美国线路居然访问不了，不慌安装完成了就没必要用这收费的VPN了，换 lantern OK)

Nice !!

继续踩坑中。。。
部署完成后设置，推送一直不成功！！提示 wrong SRC_EMAIL 错误，经过google 发现GAE后台没有设置允许推送的邮箱，于是去GAE后台设置，但是！！！

新版后台设置根本找不到推送邮件设置啊，折腾半天最后在 KindleEar github 项目的issues 里 作者给出了解答
邮箱设置 入口 （新版后台不是直接在 全局设置里，而是在 APP引擎 →设置）
https://console.cloud.google.com/appengine/settings?project=hzrkindle&moduleId=default&versionId=1
设置邮箱

推送成功了！！

完结！！
